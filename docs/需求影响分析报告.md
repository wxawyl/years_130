# 本地模型与边缘计算需求影响分析报告

## 1. 需求概述

### 1.1 核心需求
- **本地AI推理**：在移动设备端直接运行AI模型，不依赖云端API
- **数据隐私保护**：所有用户数据加密存储在本地，不上传云端
- **边缘计算**：利用设备计算能力进行实时处理
- **多平台支持**：iOS（Core ML）、Android等

### 1.2 产品愿景
打造一个以"绝对隐私"为核心卖点的健康管理应用，区别于OpenAI、豆包等云端AI服务。

---

## 2. 当前系统架构分析

### 2.1 技术栈
```
Frontend: Flutter
Backend: 无（本地应用）
Database: MemoryDatabase（内存存储）
AI Integration: OpenAI API（当前演示模式）
```

### 2.2 关键服务
- `AiIntegrationService`: 云端AI集成（OpenAI API）
- `HealthAnalysisService`: 本地规则引擎
- `SentimentAnalysisService`: 本地关键词匹配
- `DatabaseService`: 数据持久化（内存）

## 4. 风险评估

### 4.1 技术风险

| 风险项 | 风险等级 | 影响 | 缓解策略 |
|--------|---------|------|---------|
| 模型性能限制 | 🟠 中高 | 本地模型推理速度慢、精度低 | 1. 使用量化模型（4-bit/8-bit）<br>2. 模型蒸馏<br>3. 硬件加速（Core ML/NNAPI） |
| 设备兼容性 | 🟠 中 | 老旧设备不支持 | 1. 设备能力检测<br>2. 分级加载策略<br>3. 云端降级方案 |
| 模型体积过大 | 🟠 中 | 占用存储空间、下载成本高 | 1. 模型压缩<br>2. 按需下载<br>3. 增量更新 |

### 4.2 业务风险

| 风险项 | 风险等级 | 影响 | 缓解策略 |
|--------|---------|------|---------|
| 用户体验下降 | 🟠 中 | 推理速度慢、响应延迟 | 1. 离线优先设计<br>2. 渐进式加载<br>3. 缓存优化 |
| 开发成本增加 | 🟠 中高 | 需要多平台原生开发 | 1. 插件化架构<br>2. 分阶段实施<br>3. 社区开源方案 |

### 4.3 隐私安全风险

| 风险项 | 风险等级 | 影响 | 缓解策略 |
|--------|---------|------|---------|
| 加密实现错误 | 🔴 高 | 数据泄露风险 | 1. 使用成熟加密库<br>2. 安全审计<br>3. 端到端加密 |
| 模型攻击 | 🟠 中 | 模型被逆向、提取 | 1. 模型加密<br>2. 运行时保护<br>3. 定期更新 |

## 5. 技术方案建议

### 5.1 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                     Flutter UI层                         │
├─────────────────────────────────────────────────────────┤
│              应用服务层（Dart）                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │  AI 编排服务  │  │  隐私管理服务  │  │  加密服务     │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
├─────────────────────────────────────────────────────────┤
│              原生插件层（Platform Channel）              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │  iOS插件     │  │  Android插件 │  │  Web插件     │ │
│  │ (Core ML)    │  │ (TFLite)     │  │ (WASM)       │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
├─────────────────────────────────────────────────────────┤
│              本地模型层                                  │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │  量化LLM    │  │  情感分析    │  │  健康分析    │ │
│  │ (Phi-3)     │  │ 模型         │  │ 模型         │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 5.2 模型选择建议

| 场景 | 推荐模型 | 量化方案 | 精度 |
|------|---------|---------|------|
| 通用对话 | Phi-3-mini-4k | 4-bit GGUF | 良好 |
| 情感分析 | TinyBERT | INT8 | 优秀 |
| 健康建议 | Mistral-7B | 4-bit | 良好 |
| 语音识别 | Whisper-tiny | INT8 | 良好 |

### 5.3 加密方案

- **数据加密**: AES-256-GCM
- **密钥管理**: Keychain（iOS）/ Keystore（Android）
- **传输加密**: HTTPS/TLS 1.3（模型下载）

## 11. 总结与建议

### 11.1 核心结论
本需求是**战略性架构升级**，虽然开发投入较大，但将构建产品核心竞争力。建议采用**渐进式实施策略**，优先确保iOS平台体验。

### 11.2 优先级建议
1. **P0**: 数据加密 + 隐私展示（快速体现价值）
2. **P1**: iOS Core ML 基础集成 + 情感分析
3. **P2**: 完整本地LLM集成
4. **P3**: Android 支持

### 11.3 风险提示
- 本地模型能力有上限，需管理用户预期
- 需准备云端降级方案作为兜底
- 关注设备碎片化问题
